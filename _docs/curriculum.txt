Documento interno · v1.0 · Mayo 2026
Filosofia central
Programming is learned by programming. No por ver videos. No por leer explicaciones.
Por programar, equivocarse, entender por que, corregir y programar de nuevo.

Cada leccion de este curriculum sigue la formula: HOOK > CHALLENGE > THEORY > GUIDED BUILD > STRUGGLE > PROJECT > RETRIEVAL
Cada ejercicio tiene 3 niveles (Basico/Intermedio/Avanzado) con feedback pedagogico contextual, nunca solo pass/fail.
1. Principios Pedagogicos del Curriculum
Este curriculum no ensena sintaxis de Python. Ensena a pensar como programador usando Python como herramienta. Cada decision de diseno responde a uno o mas de los 8 Principios del Skillogic Method.

Principio
Aplicacion concreta en el curriculum
I. Active Learning
Nunca mas de 3 min de lectura sin escribir codigo. Cada micro-teoria va seguida de una accion inmediata.
II. Progressive Difficulty
Modelo I do > We do > You do. Las lecciones 1-4 tienen maximo scaffolding. En leccion 15+ el usuario trabaja casi solo.
III. Cognitive Load Control
Un concepto nuevo por leccion. El resto aparece como contexto ya conocido. No se mezclan conceptos nuevos simultaneamente.
IV. Retrieval Practice
Cada concepto reaparece como reto sorpresa entre los dias 3-5 post-leccion. Sin aviso. Testing effect sistematizado.
V. Interleaving
Los proyectos de consolidacion mezclan deliberadamente conceptos de multiples lecciones anteriores.
VI. Productive Failure
El CHALLENGE ocurre antes de la teoria. El usuario intenta, falla parcialmente, y su cerebro ya tiene contexto cuando llega la explicacion.
VII. Immediate Pedagogical Feedback
El sistema nunca devuelve solo 'Error linea 3'. Devuelve que fallo, por que falla, y cual es la direccion correcta.
VIII. Project-Based Consolidation
Cada modulo cierra con un proyecto real: una CLI, un analizador, un juego. Aplicacion en contexto real, no ejercicios de rellena-el-blank.

Apendice B — Reglas de Feedback Pedagogico
El feedback nunca es 'incorrecto'. Siempre contextualiza el error, explica el por que, y da la direccion correcta sin dar la solucion.

Tipo de Error
Error Raw Python
Feedback SKILLOGIC
Tipo incorrecto
TypeError: can only concatenate str (not 'int') to str
Estas intentando unir texto y numero directamente. Python no convierte automaticamente. Prueba: str(tu_numero) o f-strings.
Indice fuera de rango
IndexError: list index out of range
Tu lista tiene X elementos (indices 0 a X-1). Estas accediendo al indice Y. Imprime len(tu_lista) para verificar.
Clave no existe
KeyError: 'nombre'
La clave 'nombre' no existe en tu dict. Usa dict.get('nombre') para obtener None en lugar de error, o verifica primero con 'nombre' in dict.
Recursion infinita
RecursionError: maximum recursion depth exceeded
Tu funcion recursiva no tiene caso base, o el caso base nunca se alcanza. Dibuja el arbol de llamadas para N=3. Donde deberia detenerse?
Variable no definida
NameError: name 'x' is not defined
Python ejecuta linea a linea. 'x' se usa antes de ser definida, o esta definida en un scope diferente. Busca donde defines 'x' y si ese codigo se ejecuta antes.
Mutabilidad inesperada
Output incorrecto: lista original modificada
Las listas se pasan por referencia, no por copia. Si modificas la lista dentro de la funcion, modificas la original. Usa lista.copy() o list(lista) para crear una copia independiente.
Loop infinito
El programa no termina
Tu condicion while nunca se vuelve False. Agrega un print() dentro del loop para ver como cambia la variable de condicion en cada iteracion.
Indentacion
IndentationError o logica incorrecta por indentacion
Python usa indentacion para definir bloques. Una linea mal indentada cambia completamente que bloque pertenece. Revisa que el codigo dentro de if/for/while tenga exactamente 4 espacios mas.
